# 2026-07-08 LP5 调试配置参数说明

本文档说明当前 `tools/run_lp5_debug.py` 和 BAR06 里正在使用的 LPDDR5 调试配置参数。

当前原则：

- 正常板上调试优先使用 `tools/run_lp5_debug.py` 的默认参数。
- 脚本默认会写 BAR06 配置；因此正常流程以脚本默认值为准。
- 使用 `--skip-config` 时，脚本不会写 BAR06 配置，此时才会使用 RTL reset 默认值。
- RDC 训练配置在训练开始时锁存；训练忙时 BAR06 会挡住新的配置写入。

## 推荐命令

默认完整流程：

```powershell
python .\tools\run_lp5_debug.py --port COM4
```

默认流程会执行：

```text
外设配置 -> BAR06 配置 -> LP5 init -> RDC train -> GF
```

只跑 init 和 RDC 训练，不跑 GF：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --skip-gf
```

使用单 pattern RDC 训练：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --rdc-train-single-pattern
```

默认双 pattern RDC 训练：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --rdc-train-dual-pattern
```

RDC 训练后临时覆盖某根 DQ delay，再跑 GF：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --post-rdc-dq1-delay 70
```

## 参数优先级

建议按下面优先级理解，不要把所有参数当成同等重要。

### 常用参数

| 参数 | 什么时候改 |
| --- | --- |
| `--port` | 串口号变化时 |
| `--rdc-train-single-pattern` / `--rdc-train-dual-pattern` | 对比单 pattern 和双 pattern 训练窗口时 |
| `--rdc-train-tap-start/stop/step` | 调整 RDC 扫描范围和速度时 |
| `--post-rdc-dqN-delay` | 训练后临时偏置某根 DQ，验证 GF margin 时 |
| `--skip-gf` | 只验证 init/RDC 时 |
| `--gf-repeat` | 连续跑 GF 稳定性时 |

### 谨慎修改参数

这些参数会直接改变读写窗口，适合 ILA 配合使用。

| 参数 | 风险 |
| --- | --- |
| `--read-capture-start` | 改 INIT/RDC 读采集起点；错了会导致 RDC 全 fail |
| `--gf-capture-start` | 改 GF 读采集起点；错了会导致 GF 比较错位 |
| `--init-beat-offset` | 改 INIT/RDC 256bit burst 切片偏移 |
| `--gf-beat-offset` | 改 GF 256bit burst 切片偏移 |
| `--rd-wck-start/last` | 改 GF READ WCK 窗口 |
| `--wr-wck-start/last` | 改 GF WRITE WCK 窗口 |
| `--gf-read-done-cnt` | 改 GF READ state 持续时间；太小可能地址提前更新 |
| `--gf-write-done-cnt` | 改 GF WRITE state 持续时间；太小可能 WCK/DQ 还没收尾 |

### 一般不要动的参数

| 参数 | 说明 |
| --- | --- |
| `--baudrate` | 需要和 UART 固件一致 |
| `--command-delay` | 串口命令间隔，太小可能读写响应混乱 |
| `--init-first-read-delay` | init_start 后第一次轮询等待 |
| `--init-poll-interval` | init 轮询间隔 |
| `--post-config-delay` | 外设配置后等待 |

## BAR06 寄存器概览

BAR06 页地址是 `0x06xx`。脚本里使用完整地址，例如 `bar06_CAPTURE_CFG` 是 `0x0648`。

| 地址 | 名称 | 用途 |
| ---: | --- | --- |
| `0x0608` | `BAR06_MRW` | runtime MRW 调试命令 |
| `0x0618` | `BAR06_DQ_DELAY_L` | DQ0~DQ7 delay 写入/读回 |
| `0x0620` | `BAR06_RDC_STATUS` | 单次 RDC read check 状态 |
| `0x0624` | `BAR06_DQ_DELAY_H` | DQ8~DQ15 delay 写入/读回 |
| `0x0628` | `BAR06_RDC_TRAIN_CTRL` | RDC 训练配置 |
| `0x062C` | `BAR06_RDC_TRAIN_STATUS` | RDC 训练状态 |
| `0x0630/0x0634` | `BAR06_RDC_TRAIN_BEST_L/H` | 每根 DQ best tap |
| `0x0638/0x063C` | `BAR06_RDC_TRAIN_LEFT_L/H` | 每根 DQ window left |
| `0x0640/0x0644` | `BAR06_RDC_TRAIN_RIGHT_L/H` | 每根 DQ window right |
| `0x0648` | `BAR06_CAPTURE_CFG` | capture/WCK/GF done 计数配置 |
| `0x064C` | `BAR06_GF_STREAM_CFG` | GF bank-stream 间隔与 refresh batch |
| `0x0650` | `BAR06_GF_PATTERN_CFG` | GF 写入和期望比较图案选择 |

## BAR06_CAPTURE_CFG 0x0648

脚本通过 `pack_bar06_capture_cfg_payload()` 打包这个寄存器。

| bit | 字段 | 脚本默认 | 说明 |
| ---: | --- | ---: | --- |
| `[7:0]` | `read_capture_start_cnt` | `0x11` | INIT/MRR/RDC capture start pulse 计数 |
| `[15:8]` | `gf_capture_start_cnt` | `0x11` | GF READ capture start pulse 计数 |
| `[19:16]` | `init_beat_offset` | `0` | INIT/RDC 256bit burst 切片偏移 |
| `[23:20]` | `gf_beat_offset` | `0` | GF 256bit burst 切片偏移 |
| `[33:24]` | `gf_rd_wck_start_cnt` | `5` | GF READ WCK 开始 |
| `[43:34]` | `gf_rd_wck_last_cnt` | `25` | GF READ WCK 结束 |
| `[53:44]` | `gf_wr_wck_start_cnt` | `5` | GF WRITE WCK 开始 |
| `[63:54]` | `gf_wr_wck_last_cnt` | `18` | GF WRITE WCK 结束 |
| `[73:64]` | `gf_read_done_cnt` | `28` | GF READ state done 计数 |
| `[83:74]` | `gf_write_done_cnt` | `28` | GF WRITE state done 计数 |
| `[95:84]` | reserved | `0` | 保留 |

注意：

- `read_capture_start_cnt` 和 `gf_capture_start_cnt` 现在控制的是一拍 start pulse。
- PHY 在 start pulse 当拍采 `word0`，随后继续采 5 个 64bit word。
- `dq_a_word_valid` 标记这 6 个 64bit word 的有效周期。
- `dq_a_burst_valid` 是 256bit burst 拼好后的比较时刻。
- `init_beat_offset/gf_beat_offset` 决定从 6 个 64bit word 中取哪 4 个拼成 256bit burst。

RTL reset 默认值和脚本默认值不完全一样：

| 字段 | RTL reset 默认 | 脚本默认 |
| --- | ---: | ---: |
| `gf_rd_wck_last_cnt` | `26` | `25` |
| `gf_read_done_cnt` | `39` | `28` |
| `gf_write_done_cnt` | `27` | `28` |

正常跑脚本时，脚本会写 `BAR06_CAPTURE_CFG`，所以以脚本默认值为准。

## RX 固定采样时钟

- DQ 和 RDQS 的 ISERDES 均使用固定 0° `clk_serdes` 400 MHz。
- ISERDES `CLKDIV`、读窗口和 burst packer 继续使用核心 200 MHz。
- `BAR06 0x0614`、动态 phase FSM 和上位机 phase 参数已经删除。
- 粗粒度 burst 边界由 `init_beat_offset/gf_beat_offset` 调整，每根 DQ 的细粒度采样点由 RDC IDELAY 训练确定。
- ISERDES 只在上电或 MMCM 掉锁时由 `rst_n` 复位，运行期间不再因配置命令复位。

## BAR06_RDC_TRAIN_CTRL 0x0628

用于配置 init 后自动 RDC 训练。

| bit | 字段 | 脚本默认 | 说明 |
| ---: | --- | ---: | --- |
| `[0]` | reserved | `0` | 保留 |
| `[1]` | `apply_best` | `1` | 训练完成后是否把 best tap 写入 DQ delay |
| `[2]` | `init_enable` | `1` | init 结束后是否自动执行 RDC train |
| `[3]` | reserved | `0` | 保留 |
| `[7:4]` | `dq_start` | `0` | 起始 DQ |
| `[11:8]` | `dq_end` | `15` | 结束 DQ |
| `[20:12]` | `tap_start` | `0` | tap 扫描起点 |
| `[29:21]` | `tap_stop` | `300` | tap 扫描终点 |
| `[38:30]` | `tap_step` | `2` | tap 扫描步进 |
| `[39]` | `dual_pattern` | `1` | `1` 双 pattern，`0` 单 pattern |
| `[95:40]` | reserved | `0` | 保留 |

当前 pattern 定义：

```text
dual_pattern = 0:
    只跑 pattern0: MR33=0x5A, MR34=0xA5

dual_pattern = 1:
    先跑 pattern0: MR33=0x5A, MR34=0xA5
    再跑 pattern1: MR33=0x3C, MR34=0xC3
    两次 err_bitmap OR 到一起，只有两次都 pass 的 tap 才算 pass
```

每个 MR 保存 8 个 beat，MR33/MR34 合起来形成一个 BL16 时间图案。
每个 beat 的 DQ 期望使用 `0x5555` 或 `0xAAAA`，保证相邻 DQ 电平相反。

脚本参数：

| 参数 | 说明 |
| --- | --- |
| `--rdc-train` | 开启 RDC 训练，默认开启 |
| `--no-rdc-train` | 关闭 RDC 训练 |
| `--rdc-train-dq-start N` | 起始 DQ |
| `--rdc-train-dq-end N` | 结束 DQ |
| `--rdc-train-tap-start N` | 起始 tap |
| `--rdc-train-tap-stop N` | 结束 tap |
| `--rdc-train-tap-step N` | 步进 |
| `--rdc-train-no-apply-best` | 只扫描窗口，不把 best tap 写回 DQ delay |
| `--rdc-train-dual-pattern` | 双 pattern，默认 |
| `--rdc-train-single-pattern` | 单 pattern |

训练时间估算：

```text
单 pattern 时间 约等于 tap_count * 1 组 RDC read
双 pattern 时间 约等于 tap_count * 2 组 RDC read
```

当前默认 `tap_start=0, tap_stop=300, tap_step=2`：

```text
tap_count = floor((300 - 0) / 2) + 1 = 151
单 pattern: 151 组 RDC read
双 pattern: 302 组 RDC read
```

## BAR06_RDC_TRAIN_STATUS 0x062C

脚本轮询这个寄存器判断训练是否完成。

| bit | 字段 | 说明 |
| ---: | --- | --- |
| `[0]` | busy | 训练忙 |
| `[1]` | done | 训练完成 |
| `[2]` | pass_all | 目标 DQ 范围全部找到窗口 |
| `[3]` | init_done | init 已完成或训练器见过 init ready |
| `[12:4]` | reserved | 保留 |
| `[16:13]` | state | `rdc_train` 当前状态 |
| `[20:17]` | dq_start | 当前/配置起始 DQ |
| `[29:21]` | tap | 当前 tap |
| `[39:30]` | best_len | 当前 DQ 的 best window 长度 |
| `[55:40]` | last_err_bitmap | 最近一次 tap 的错误 bitmap |
| `[71:56]` | pass_mask | 训练通过的 DQ |
| `[87:72]` | fail_mask | 训练失败的 DQ |
| `[95:88]` | reserved | 保留 |

## DQ delay 配置和读回

寄存器：

```text
BAR06_DQ_DELAY_L 0x0618: DQ0~DQ7
BAR06_DQ_DELAY_H 0x0624: DQ8~DQ15
```

格式：

```text
每根 DQ 9bit
DQ0 = bits [8:0]
DQ1 = bits [17:9]
...
```

当前 RTL reset 默认所有 DQ delay 为 `0`。

正常流程下不建议启动前手动写 DQ delay，推荐让 RDC train 自己扫。

注意：

- RDC train 开始时会从默认 delay 重新开始扫描。
- 因此 `--dqN-delay` 不适合和默认 RDC train 流程一起用来固定最终 delay。
- 想验证某根 DQ 的 GF margin，应该使用 `--post-rdc-dqN-delay`，让脚本在 RDC train 完成后再覆盖该 DQ。

两种调试入口：

```powershell
# init 前写 DQ1 delay；默认 RDC train 会重新清零扫描，通常少用
python .\tools\run_lp5_debug.py --port COM4 --dq1-delay 30

# RDC train 完成后覆盖 DQ1，再跑 GF，常用于验证 GF margin
python .\tools\run_lp5_debug.py --port COM4 --post-rdc-dq1-delay 70
```

## GF timing 参数

这些参数进入 `lpddr5_gf_engine.v`。

| 参数 | 作用 |
| --- | --- |
| `gf_rd_wck_start_cnt` | GF READ 状态中 WCK 开始翻转的计数 |
| `gf_rd_wck_last_cnt` | GF READ 状态中 WCK 停止翻转的计数 |
| `gf_wr_wck_start_cnt` | GF WRITE 状态中 WCK 开始翻转的计数 |
| `gf_wr_wck_last_cnt` | GF WRITE 状态中 WCK 停止翻转的计数 |
| `gf_read_done_cnt` | GF READ 状态结束计数 |
| `gf_write_done_cnt` | GF WRITE 状态结束计数 |

当前 GF engine 内部写数据有效窗口：

```text
GF_WR_DATA_START_CNT = 0
GF_WR_DATA_LAST_CNT  = 26
```

这表示 `wave_dq_a_tx_word` 在 `cnt_write=0..26` 期间持续更新并打开 DQ 输出。

WCK 窗口是另一套配置：

```text
WRITE WCK: gf_wr_wck_start_cnt .. gf_wr_wck_last_cnt
READ  WCK: gf_rd_wck_start_cnt .. gf_rd_wck_last_cnt
```

因此：

- 写数据持续时间不等于 WCK 持续时间。
- 改 WCK start/last 会改变 die 看到 DQ 的有效采样窗口。
- 改 done_cnt 会改变 GF FSM 什么时候允许进入下一笔访问。

当前建议：

```text
gf_read_done_cnt  = 28 已经是当前提速调试里验证过的较短值
gf_write_done_cnt = 28 当前脚本默认值
rd_wck_last       = 25 是脚本默认值；若本地实验要继续放宽可用 --rd-wck-last 覆盖
```

## runtime MRW

`BAR06_MRW 0x0608` 用于 runtime MRW 调试。

脚本参数：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --mrw 21:0xaa
```

含义：

```text
MR21 = 0xaa
```

注意：

- runtime MRW 应在 init/RDC/GF 空闲时使用。
- RDC train 期间，训练器会自己发 MR33/MR34/MRR8，不建议手动插 MRW。

## 参数和 ILA 信号对应关系

常用观察关系：

| 参数/状态 | ILA 信号 |
| --- | --- |
| INIT/RDC/GF 采集 start pulse | `lpddr5_dut1/channel_rx_dq_capture_en` |
| 原始每周期采集 word | `lpddr5_dut1/dbg_dq_word_raw` |
| 256bit burst 拼接完成 | `lpddr5_dut1/dq_a_burst_valid` |
| 256bit burst 内容 | `lpddr5_dut1/dq_a_burst_flat` |
| GF 当前读计数 | `lpddr5_dut1/gf_cnt_read` |
| GF 当前写计数 | `lpddr5_dut1/gf_cnt_write` |
| GF DQ 输出使能 | `lpddr5_dut1/channel_dq_oe` |
| GF 写出 64bit word | `lpddr5_dut1/channel_dq_a_tx_word` |
| GF 比较窗口 | `lpddr5_dut1/dbg_gf_compare_window` |
| GF odd/even mismatch | `lpddr5_dut1/dbg_gf_compare_mismatch_odd/even` |

## 调参建议

1. 先固定脚本默认值，确认 RDC train pass 和 GF 是否稳定。
2. 若 RDC 窗口异常，先看 `read_capture_start`、beat offset 和每根 DQ 的训练窗口。
3. 若 RDC pass 但 GF 单 bit 错，先用 `--post-rdc-dqN-delay` 验证是不是读 margin。
4. 若 GF 后半段错，配合 ILA 看 `gf_cnt_write`、`channel_dq_a_tx_word`、`dq_a_burst_flat`，再决定扫 WCK 或 done_cnt。
5. 要比较 RDC 图案强度时，用 `--rdc-train-single-pattern` 和默认双 pattern 对照。

当前比较稳的基线命令：

```powershell
python .\tools\run_lp5_debug.py --port COM4
```

如果只想看 RDC 训练窗口：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --skip-gf
```

如果要验证 DQ1 GF margin：

```powershell
python .\tools\run_lp5_debug.py --port COM4 --post-rdc-dq1-delay 70
```
